# Contributing

We welcome contributions to the Angular 21 Admin Template!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/angular21-admin.git`
3. Install dependencies: `npm install`
4. Create a feature branch: `git checkout -b feature/your-feature`

## Development Workflow

### Development Server
```bash
npm start
```

### Build
```bash
npm run build
```

### Testing
```bash
npm test
```

### Linting
```bash
npm run lint
```

## Code Style

- Follow Angular style guide
- Use TypeScript strict mode
- Use single quotes for strings
- Use 2-space indentation
- Add comments for complex logic
- Write unit tests for new functionality

## Commit Guidelines

Follow the [Conventional Commits](https://www.conventionalcommits.org/) format:

- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `style`: Code style changes (formatting, missing semicolons, etc.)
- `refactor`: Code refactoring
- `test`: Adding or updating tests
- `chore`: Build or tooling changes

Example:
```
feat(factory): add factory list page
fix(auth): fix login authentication issue
docs(readme): update project structure
```

## Pull Request Guidelines

1. Create a descriptive title
2. Include a summary of changes
3. Reference related issues
4. Ensure tests pass
5. Ensure linting passes
